## 2. 建议同时确认的运行基线

以下是方案已提出但还需要产品/部署侧确认的内容。它们不改变主架构，却会影响开发环境和验收方式。

| 主题 | 建议默认值 | 需要你的输入 |
| --- | --- | --- |
| 若依版本 | 官方 `RuoYi-Vue v3.9.2`，Spring Boot 4.0.6、Java 17+。 | 采纳或指定其他发行版/版本。 |
| 部署形态 | 单域名反向代理，Next.js + RuoYi + MySQL + Redis；本地 MinIO 用于文件。 | 目标服务器、域名、是否已有 MySQL/Redis/对象存储。 |
| 搜索 | 首期数据库搜索与前端高亮；质量或规模要求提高时接入专用搜索服务。 | 是否必须首期使用 Elasticsearch/OpenSearch/其他服务。 |
| 移动端 | 用户端/阅读器响应式；作者端、运营端以桌面优先。 | 是否需指定手机浏览器与最低版本。 |
| 统计 | 初期按事件记录加定时聚合实现。 | 追读率、留存、热榜、热搜、渠道和收入的正式口径。 |
| 外部登录 | 独立认证子域，需短信/邮箱和微信/QQ OAuth 凭据。 | 首期是否已有供应商账号和回调域名。 |
| Qwen 审核 | Spring AI 2.0.0，百炼 OpenAI 兼容接口；模型仅由部署参数指定。 | 工作空间、地域、模型、`DASHSCOPE_API_KEY`、超时和配额。 |
| BFF 会话 | Redis 保存 BFF 到若依 Token 的映射，浏览器只保存 HttpOnly 会话标识。 | Cookie 域名、HTTPS、CSRF 密钥和 Redis 前缀。 |
